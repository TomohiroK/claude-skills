# 会話クローズ時の必須フロー

## 第一原則: 経験はルールに落とさなければ消える

feedback メモリは会話内でのみ効力を持つ。会話をまたいだ時点で、メモリに書いた反省は次の自分には届かない。
ルール（`.claude/rules/`）に書いて初めて、全エージェント・全会話で強制力を持つ。

## 会話クローズ時の必須手順

ユーザーから「クローズして」「終わり」「反省会」等の指示を受けたら、以下を**必ず実行する**。

### Step 1: feedback メモリの棚卸し
- MEMORY.md の feedback セクションを全件読む
- 各 feedback メモリの内容を確認する

### Step 2: ルール化状況の確認
- 各 feedback が `.claude/rules/` のいずれかのルールファイルに反映されているか確認する
- 確認方法: feedback の核心（ルール本体）を grep で `.claude/rules/` 内から検索する

### Step 3: 未ルール化の feedback をルールに落とす
- ルール化されていない feedback を、適切なルールファイルに追記する
  - 既存ルールのスコープに合致 → 既存ファイルにセクション追加
  - 新しい領域 → 新規ルールファイルを作成
- ルールには以下を含める:
  - 何をすべきか / すべきでないか（明確な指示）
  - なぜそのルールが必要か（発生事例）
  - どの場面で適用するか（トリガー条件）

### Step 4: 確認報告
- ルール化した件数と対象をユーザーに報告する
- 「全 feedback がルール化済み」であることを明示する

## 禁止事項

- 会話上で「反省しました」「次回から気をつけます」と言うだけで終わること — ルールファイルに書かなければ無意味
- feedback メモリに書いて「記録しました」で満足すること — メモリは次の会話に届く保証がない
- 「軽微な feedback だからルール化不要」と判断すること — 軽微かどうかは100回の実績で決まる（no-shortcuts.md 参照）

## このルール自体の根拠

2026-03-30、スケジュールタスクのインシデント対応後、会話内で反省点を述べたがルールファイルには書かなかった。ユーザーから「ここで反省っぽいことをしても意味ないですよ？」「メモリにある反省材料は全部何らかのルールに落とさないと、せっかくの経験が落ちます」と指摘。
