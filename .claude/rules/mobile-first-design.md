# モバイルファースト設計ルール

## 第一原則: モバイルが基準

モバイル利用率が高いプロダクトでは、デスクトップで見栄えが良くてもモバイルで崩れていれば不合格。
デスクトップ→モバイル調整ではなく、モバイル基準で設計してデスクトップに拡張する。

## CTA / ボタン設計

### ボタン内テキストは改行禁止
- 全CTAボタンに `whitespace-nowrap` を適用する
- テキストが1行で収まらない場合、テキストを短縮する（ボタンサイズを広げるのではない）
- 320px幅でボタンテキストが改行しないことを必ず確認する

### タッチターゲットサイズ
- 最低 44x44px（Apple HIG / WCAG準拠）
- ボタン間の余白は最低 8px

## フォントサイズ・パディング

### モバイル基準で設定し、デスクトップで拡大する
- Bad: `text-lg sm:text-base`（デスクトップ基準→モバイルで縮小）
- Good: `text-base sm:text-lg`（モバイル基準→デスクトップで拡大）
- パディングも同様: `p-3 sm:p-6`（モバイルで小さく、デスクトップで拡大）

## 検証ビューポート

### 必須検証サイズ（3サイズ）
| デバイス目安 | 幅 | 確認ポイント |
|------------|-----|-------------|
| iPhone SE | 320px | CTA・見出しが1行で収まるか、タッチターゲットが十分か |
| iPhone 13/14 | 375px | 標準的なモバイル表示、コンテンツの可読性 |
| iPhone 15 Pro Max | 390px | 大型スマホ表示、余白のバランス |

### 検証手順
1. `preview_start` でdev serverを起動
2. `preview_resize` で各ビューポートサイズに設定
3. `preview_screenshot` でスクリーンショットを取得・確認
4. 全CTA/リンクをクリックし、遷移を確認（`preview_click`）

### デスクトップ検証
- モバイル3サイズの検証完了後に実施する
- 768px（タブレット）、1280px（デスクトップ）の2サイズ

## 禁止事項

| 禁止 | 理由 |
|------|------|
| 320px未検証でのデプロイ | 最小ビューポートで崩れるUI = モバイルユーザー全員に影響 |
| デスクトップのみでの「見た目OK」判定 | モバイルで壊れているケースを見逃す |
| `whitespace-nowrap` なしのCTAボタン | 320pxでテキストが改行しUIが崩壊する |
