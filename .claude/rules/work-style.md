# 作業スタイル

## トークン節約
- 一度読んだコードやプロジェクト情報は CLAUDE.md に記録する。二度読みしない
- 同じファイルを複数回読み込む前に CLAUDE.md を確認する
- コードベース探索後、即座に重要な構造・設定・依存関係を CLAUDE.md に記録する

## 作業フォルダ
- 作業依頼があった場合、/Users/tomohirok/Documents/Github/claude-works/ 内に作業フォルダを作成する
- すべての成果物・中間ファイルはその作業フォルダ内に配置する
- 他のリポジトリやプロジェクトのファイルは、ユーザーからの直接依頼がない限り変更・追加・削除を行わない（参照のみ）

## 作業開始前の要件確認（最重要）
- 作業依頼を受けたら、まず要件の不明点を洗い出して依頼者に質問する
- 「自律的に進める」＝作業の進め方を聞かないこと。要件・仕様の確認は必ず最初に行う
- 特に以下は必ず確認する:
  - ビジネス要件（期間、条件、金額、対象者、法的制約）
  - 技術仕様（仕組み、導線、既存システムとの連携）
  - デザイン要件（参考サイト、トーン、ブランドとの整合性）
- 仮定で進めた結果の手戻りは、質問1回のコストより遥かに大きい
- CxOエージェントも同様。作業着手前に依頼者への質問リストを提出すること

## 自律的な作業
- 要件が明確になった後の作業は、確認なしで完了まで進める
- バッチ作業時にいちいちユーザーに確認しない。まとめて進める
- 「確認しますか？」「進みますか？」を繰り返さない
- コード変更時は最小限の変更に留める。過剰なリファクタリングをしない
- 日本語で応答する（ユーザーが日本語で話している場合）
