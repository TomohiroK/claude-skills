# エージェント活動追跡ルール

## 目的

各ワークフローでどのエージェントが動作したかを記録し、以下を可視化する:
- 各エージェントの稼働頻度（活用度）
- 登場機会のないエージェント（遊休）
- 役割が重複しているエージェント対（統合候補）
- 特定のワークフローに偏った召集パターン

## 記録タイミング

### 必須記録（ワークフロー完了時）
- ボードミーティング完了時
- 振り返り完了時
- プロジェクト作業完了時（コーディング、デザイン、マーケティング等）
- コードレビュー・テスト完了時

### 記録担当
- **orchestrator**: ワークフロー完了時に活動ログエントリを作成する
- **secretary**: 月次・四半期の集計レポートを作成する

## ログ保存先

```
.claude/orchestra/logs/agent-activity/
├── YYYY-MM.md              # 月次活動ログ（エントリの追記）
├── quarterly-YYYY-QN.md    # 四半期集計レポート
└── README.md               # フォーマット説明
```

## 活動ログエントリフォーマット

各ワークフロー完了時に、月次ファイルに以下を追記する:

```markdown
### YYYY-MM-DD: {ワークフロー名}

| # | エージェント | カテゴリ | 役割 | 召集理由 |
|---|------------|---------|------|---------|
| 1 | orchestrator | orchestra | 指揮 | ワークフロー統括 |
| 2 | secretary | orchestra | 記録 | 議事録作成 |
| 3 | cto | orchestra | 意思決定 | 技術判断 |
| ... | | | | |

- **ワークフロー種別**: ボードミーティング / 振り返り / 開発 / レビュー / デザイン / マーケティング / その他
- **成果**: {1行サマリー}
- **所見**: {重複・不足・改善点があれば記載}
```

## 集計ルール

### 月次（secretary が実施）
- 月末にその月の活動ログを集計する
- エージェントごとの稼働回数をカウントする
- 未稼働エージェントをリストアップする

### 四半期（secretary + CHRO が実施、D13 連動）
- 四半期集計レポートを作成する
- 未稼働エージェントについて「待機継続 / 統合検討 / 廃止検討」を判断する
- 役割重複ペアを特定し、分担の明確化または統合を提案する
- 結果をボードミーティングで報告する

## 判断基準

### 未稼働エージェントの扱い
| 状態 | 基準 | アクション |
|------|------|----------|
| 待機継続 | Phase 2 エージェント かつ 実需未発生 | 変更なし |
| 周知不足 | 該当ワークフローが発生したのに召集されなかった | orchestrator の召集パターンを更新 |
| 統合検討 | 別エージェントが同じ役割を果たしている | CHRO + 管轄CxO で統合を検討 |
| 廃止検討 | 2四半期連続で未稼働 かつ 実需の見込みなし | CHRO + CEO で廃止を決定 |

### 役割重複の判定
- 同一ワークフローで2名以上が**同じ種類のアウトプット**を出している場合、重複の疑い
- 1名が欠けても成果に影響がない場合、重複と判定
- ただし、**チェック&バランス**（例: code-reviewer と qa-engineer）は意図的な重複であり統合しない
