# プロジェクト作業の事前確認

- 新しいプロジェクトに触れる際は、まずビルドツール設定を確認する（Trunk.toml, Cargo.toml, package.json, launch.json等）
- 類似名ディレクトリが存在する場合、作業対象を最初に確認する
- 静的ファイル配置時はビルドパイプラインのコピー設定を必ず確認する（Trunkのcopy-file/copy-dir等）

## エラー発生時のデバッグ手順（必須）

1. エラー内容を正確に記録する（エラーメッセージ、スタックトレース、発生条件）
2. **最初に自分の変更を疑う**: `git diff` で直近の変更を確認する
3. **変更を一旦戻す**: `git stash` → エラーが再現するか確認する
   - 再現しない → 自分の変更が原因。差分を細かく分割して原因特定
   - 再現する → 環境要因。キャッシュクリア、依存の再インストール等
4. **3回ルール**: 同じアプローチ（キャッシュクリア、再起動等）を3回繰り返しても解決しない場合、そのアプローチを中止し、別のアプローチに切り替える

禁止事項:
- エラーの原因を環境・ツールのせいと決めつけること（確証バイアスの排除）
- キャッシュクリアと再起動の無限ループ（「キャッシュクリアは祈りではない」）
