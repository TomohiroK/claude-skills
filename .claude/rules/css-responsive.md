# CSS / レスポンシブ設計ルール

## 装飾用オーバーレイの pointer-events

### 装飾要素には必ず `pointer-events-none` を付与する
- absolute / fixed 配置の装飾要素（グラデーション、背景エフェクト、シャドウ等）は `pointer-events-none` を必須とする
- この要素がインタラクティブ要素（リンク、ボタン、入力欄）と重なると、クリック/タッチが装飾要素に奪われる
- デスクトップで問題なくてもモバイルで発生する（画面が小さいほど要素の重なりが起きやすい）

```html
<!-- Good: 装飾要素に pointer-events-none -->
<div class="absolute inset-x-0 bottom-0 h-1/3 bg-gradient-to-t pointer-events-none"></div>

<!-- Bad: pointer-events-none なし。下のボタンがクリック不能になる可能性 -->
<div class="absolute inset-x-0 bottom-0 h-1/3 bg-gradient-to-t"></div>
```

### 例外: クリック可能な装飾要素
- オーバーレイ自体がクリック可能（モーダルの背景クリックで閉じる等）な場合は `pointer-events-auto` を明示する
- その場合でも、オーバーレイ内のインタラクティブ要素が意図通りクリック可能であることを検証する

## レスポンシブ設計時の重なり検証

### 最小ビューポートでの z 軸確認
- レスポンシブデザインでは、ビューポートが縮小すると要素の重なり方が変わる
- 特に以下の組み合わせは注意:
  - absolute/fixed 要素 + インタラクティブ要素（リンク、ボタン）
  - 高さが相対値（h-1/3, h-1/2 等）の要素 + 固定高さのコンテンツ
  - ヒーローセクション内の CTA ボタン + 装飾オーバーレイ

### 検証手順
1. DevTools で 375px 幅に設定
2. 全 CTA / リンクをクリックし、遷移を確認
3. Elements パネルで absolute/fixed 要素をホバーし、インタラクティブ要素との重なりを確認

## DOM順序とスタッキングコンテキスト

### DOM順序の後ろにある要素は上に描画される
- 同じスタッキングコンテキスト内で z-index を指定しない場合、DOM順序が後の要素が上に描画される
- 装飾要素をコンテンツの後に配置する場合、`pointer-events-none` がないとクリックを奪う

### 安全な配置パターン
```
パターンA: pointer-events-none（推奨）
  コンテンツ div
  装飾 div (pointer-events-none)  ← DOM順序で上に来るが、クリックを透過する

パターンB: z-index で明示的に制御
  コンテンツ div (z-10)  ← z-index で上に
  装飾 div (z-0)         ← z-index で下に

パターンC: DOM順序を入れ替える
  装飾 div               ← 先に描画（下に来る）
  コンテンツ div          ← 後に描画（上に来る）
```
- パターンA が最もシンプルで安全。パターンB/C は z-index 管理の複雑さが増す

## モバイルQA必須項目

リリース前に以下を確認する:

| # | 確認項目 | ビューポート |
|---|---------|------------|
| 1 | 全CTA/リンクのクリック遷移が正常 | 375px, 768px |
| 2 | フォーム入力欄のフォーカス・入力が可能 | 375px |
| 3 | スクロールが正常（固定要素にブロックされない） | 375px |
| 4 | absolute/fixed 装飾要素がインタラクティブ要素を遮蔽していない | 375px |
| 5 | タッチターゲットが十分な大きさ（最低44x44px） | 375px |

## 発生事例

### 3x3 Observer's Hub ヒーローCTAバグ（2026-03-26）
- 装飾用グラデーション（`absolute inset-x-0 bottom-0 h-1/3`）に `pointer-events-none` が未設定
- モバイルでヒーロー高さが約211pxに縮小し、CTAボタンがグラデーション領域（下1/3）と重なった
- 修正: `pointer-events-none` を1行追加
- 教訓: 装飾要素は pointer-events-none。デスクトップで動いてもモバイルで壊れる
