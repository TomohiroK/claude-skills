# 画像加工・バナー制作ルール

## 第一原則: 使えないものを出すな

「バナー」「ビジュアル」として納品する以上、以下の**全て**を満たすこと：
- テキストが全て読める
- 人物が意図通り見える
- 訴求要素（金額、特典、CTA等）が一目で分かる

いずれか1つでも欠けたら**ボツ**。サイズ数を揃えるより1つの完成品を出す方が価値がある。

## 画像ファイルサイズ（必須チェック）

Web用画像の加工は**圧縮から始める**。レイアウトやクロップの前に圧縮。

| 用途 | 上限 | 推奨 |
|------|------|------|
| Web表示 | 300KB | 100-200KB |
| モバイルバナー | 100KB | 50-80KB |
| OGP/SNS | 300KB | 150-250KB |

### 圧縮手順（macOS）
```bash
# 1. サイズ確認
ls -lh image.png

# 2. リサイズ + JPEG変換（sips）
sips -Z 2000 image.png --out optimized/image-2000w.jpg -s format jpeg -s formatOptions 80
sips -Z 1200 image.png --out optimized/image-1200w.jpg -s format jpeg -s formatOptions 80
sips -Z 800  image.png --out optimized/image-800w.jpg  -s format jpeg -s formatOptions 75

# 3. 結果確認
ls -lh optimized/
```

**禁止**: MB単位の画像をHTMLに埋め込む。「後で圧縮する」は禁止。最初に圧縮する。

## アスペクト比の互換性

元画像のサイズバリエーションを作る前に、元画像のアスペクト比と出力先のアスペクト比を比較する。

### 判定基準
- 元画像と出力先の比率が近い（±50%以内）→ リサイズ/クロップで対応可能
- 比率が大幅に異なる → `object-fit: cover` のクロップでは主要要素が欠ける → **そのサイズは作らない**

### 作成禁止の例
| 元画像 | 出力先 | 判定 | 理由 |
|--------|--------|------|------|
| 4:1 横長 | 幅違い横長 | OK | 比率が同じ。全要素が見える |
| 4:1 横長 | 1:1 正方形 | NG | 左右75%カット。テキストも人物も消える |
| 4:1 横長 | 9:16 縦長 | NG | ほぼ全てが見えない |
| 4:1 横長 | 300x250 | NG | 主要テキストが読めない |

### 異なる比率が必要な場合
元画像のクロップではなく、**HTML/CSSで背景+テキストをレイアウトし直す**。
その場合も「全要素が読める」が完了条件。

## 発生事例（2026-03-31）

歯科衛生士お友達紹介キャンペーンのバナー加工:
- 8MB PNG を圧縮せず使用 → ページが重くて動かない
- 4:1横長画像を正方形・縦長・スカイスクレイパー等8サイズに `object-fit: cover` でクロップ
- 全てのサイズでテキストが読めない・人物が切れる・金額が見えない
- 「使えもしないもん出してくんな」と指摘
- 修正: 圧縮 (7.6MB→60-267KB) + 元画像の横長比率を活かした幅違い5種に絞って解決
