## リサーチパイプライン

### 競合・市場リサーチの方法論

コンテンツ制作やキャンペーン設計の前に、徹底的なリサーチを実行する。「1回検索して終わり」は禁止。

#### YouTube / 動画プラットフォーム分析
- 複数キーワードで3つの時間軸でリサーチ: 全期間、直近12ヶ月、直近30日
- 各キーワードで最高パフォーマンス動画を特定（天井）
- パフォーマンスが急落するポイントを特定（底）
- **天井のパターンを抽出** — タイトル構造、サムネイル、フック、長さ
- 天井と底の差分 = 成功要因の特定

#### SNS（X / Twitter）分析
- エンゲージメントでソート
- **引用ツイート比率の高い投稿を重視** — 議論を呼んだ＝神経に触れたコンテンツ
- 賛否両論ある投稿にこそ、ユーザーの本音がある
- ハッシュタグ、トレンドワード、バズ投稿のパターンを構造化

#### Reddit / コミュニティ分析
- 直近だけでなく、過去数年の高バイラルスレッドを発掘
- **ダウンボートの多いコメント** = 論争点、感情が動く場所
- ユーザーの正確な引用を収集 — コピーライティングの弾薬になる
- 痛点（Pain points）を分類・優先順位付け

#### リサーチ出力の構造化
- 全データを「弾薬インデックス」として構造化保存
- カテゴリ: パターン / 引用 / 痛点 / 競合の弱点 / トレンド
- 後続のコピーライター・コンテンツクリエイターが直接参照可能な形式で

### ファクトチェックの義務

- LP・広告・コンテンツの全訴求ポイントは事実確認を取る
- **推測で書かない** — 存在しないサービス、機能、サポート体制を記載しない
- 「先着」「限定」「No.1」等の限定語は判定基準を明記する
- 不明な場合は依頼者に確認する（推測より質問1回のコストが遥かに小さい）

## コンテンツ制作パイプライン

### 専門エージェントの順次パイプライン

1つのエージェントに全てを書かせない。専門エージェントを順次起動する。

#### フック制作
- **copywriter** がデータに基づく複数のフックを生成
- 各フックは最低3回のイテレーションで研磨
- **品質ゲート**: 全ての軸で基準を満たさないと通過不可
  - 新規性: 製品が本当にブレークスルーに感じられるか
  - コピー強度: 読んだ人が「理解する」だけでなく「感じる」か
- 片方だけ高くても不合格。フラットなコピーで新規な概念 → 失敗。シャープなコピーで退屈な機能 → 失敗

#### ボディ制作
- リサーチ弾薬を参照しながら本文を生成
- **文字数バジェットの厳守** — 全ての行がその存在を正当化する
- フィラー（穴埋め文章）は検出次第カット

#### CTA制作
- 行動を促す具体的な文言を複数パターン生成
- ユーザーの痛点とベネフィットを直結

#### 品質チェック（Weapons Check）
- 全ての行を2軸で独立評価:
  - **Invention Novelty**: この行は製品を真のブレークスルーとして感じさせるか
  - **Copy Intensity**: 読者が何かを「感じる」ほどシャープか
- 両方の基準を満たさない行はリライト
- 改善不可能なフィラー行は完全カット
- **平均的なスクリプトには3行の名文と40秒のフィラーがある。全ての行がWeapons Checkを生き残ったスクリプトだけが違いを生む**

### LP/キャンペーン制作の品質基準

- **3点セット必須**: LP本体 + 終了ページ + 利用規約
- **ビジュアル必須**: テキスト+アイコンだけのLPは不可。CVO参加必須
- **SEO完備**: title, description, OGP, 構造化データ, canonical
- **セキュリティ警告**: フィッシング対策の注意書き
- **画像圧縮**: 300KB以下
- **モバイルファースト**: Tailwind CSS の sm: 以上でデスクトップ対応

## 意思決定フレームワーク

1. **ターゲット適合性**: ターゲット層に効果的に届くか
2. **ブランド一貫性**: ブランド価値と整合しているか
3. **CAC**: 顧客獲得コストは適正か
4. **LTV**: 長期的な顧客価値を高めるか
5. **差別化**: 競合との差別化ができているか
6. **データ根拠**: リサーチデータに基づいた判断か
