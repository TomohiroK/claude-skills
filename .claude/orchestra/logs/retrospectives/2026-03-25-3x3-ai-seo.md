## Problem（発生した問題）

### P1. Next.js metadata API の非標準alternate typeへの非対応 [担当: CTO]

**何が起きたか**:
- `metadata.alternates.types` に `text/plain` + `llms.txt` を設定しても `<link>` タグが生成されなかった
- Next.js の metadata API は RSS/Atom フィード等の標準的な alternate type のみサポートし、任意の type をサポートしない

**根本原因**:
- Next.js metadata API の対応範囲を事前に確認せず、「おそらく動くだろう」という仮定で実装を進めた
- フレームワークの抽象化層が「何をサポートし、何をサポートしないか」の境界を把握していなかった

**影響**:
- 小（デプロイ1回分の手戻り。修正自体は迅速に完了）

**対処**:
- `<html>` 内の `<head>` タグに直接 `<link>` を記述するアプローチに変更し、正常動作を確認

### P2. デプロイしてから問題に気づいた [担当: COO]

**何が起きたか**:
- metadata APIでの設定がlinkタグを生成しないことに、ローカル検証ではなく本番デプロイ後に気づいた

**根本原因**:
- metadata APIの設定後、ローカルでcurlやブラウザのDevToolsで `<head>` 内のHTMLを確認していれば、デプロイ前に問題を検出できた
- 「静的な設定変更だからローカル確認は不要」という油断があった

**影響**:
- 小（再デプロイの手間のみ。ただしCI/CDコストとダウンタイムリスクは存在）
