## Problem（発生した問題）

### P1. 事前同意が口頭のみ [担当: CLO] -- 重大
- **何が起きたか**: ペネトレーションテストの許可を口頭（チャット）のみで取得。書面での合意なし
- **リスク**: テスト中にインシデントが発生した場合、法的保護が不十分。不正アクセス禁止法との関係が曖昧
- **改善策**: 今後のセキュリティ監査では必ず事前同意書（テスト範囲・期間・責任範囲・免責事項を含む）を取得する

### P2. テスト用データの本番DB残存リスク [担当: CTO/CLO]
- **何が起きたか**: Firebase Authで作成したテストアカウントは即削除したが、GCP Cloud Loggingにテストのログ（IPアドレス、操作履歴）が残存している可能性がある
- **改善策**: テスト完了後、管理者にFirebase側のログ確認・クリーンアップを依頼するプロセスを標準化

### P3. レポートに機密情報を平文記載 [担当: CLO]
- **何が起きたか**: Firebase APIキー、プロジェクトID、GTM ID等の実際の値をレポートに記載
- **リスク**: レポートが漏洩した場合、攻撃者に情報を提供することになる
- **改善策**: レポートの保管には暗号化・アクセス制限を適用。外部共有時はマスキング版を作成

### P4. 認証後の検査が未実施 [担当: CTO]
- **何が起きたか**: アカウント作成に成功したにもかかわらず、認証後のセッションで水平/垂直権限昇格テストを実施していない
- **リスク**: Firebase Security Rules の不備やroleフィールド改ざんによる管理者権限取得の可能性を見逃した
- **改善策**: 標準チェック項目リストに認証後テスト（Phase 3-4）を必須項目として組み込む

### P5. セキュリティ専任エージェントの不在 [担当: CTO/CHRO]
- **何が起きたか**: セキュリティ監査を汎用的なフローで実施。専任エージェントも標準チェックリストもなく、テスト項目に漏れが発生
- **改善策**: testing/ カテゴリに security-auditor（外部監査）と security-reviewer（コードレビュー）の2名を新設する

### P6. サプライチェーン検査の欠如 [担当: CTO]
- **何が起きたか**: Next.js 14.0.21の既知CVE確認、依存パッケージの脆弱性スキャンが未実施
- **リスク**: Server Actions関連の脆弱性が14.x系で報告されているが、該当するか未確認
- **改善策**: Phase 6（サプライチェーン検査）を標準チェック項目に組み込む
