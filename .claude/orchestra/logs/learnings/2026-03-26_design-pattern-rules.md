# 学習事項: デザインパターンルールのレビュー（2026-03-26）

## 学習1: MobbinリサーチはB2Cに偏る
- **状況**: Mobbinの収録アプリはコンシューマー向けが大半
- **影響**: データテーブル、B2B認証（SSO/SAML）、管理者画面など、B2B SaaSの主要パターンが漏れた
- **対策**: B2B SaaSのパターンは別途リサーチソース（SaaS Design Patterns、Enterprise UX等）で補完する
- **ルール化**: ui-design-patterns.md にB2Bカテゴリを追加する際、Mobbinだけでなくドメイン固有のリサーチを実施すること

## 学習2: ローカライゼーションはデザインシステムの土台
- **状況**: 多言語対応（数値・日付・通貨フォーマット、フォントフォールバック）がトークン定義に含まれていなかった
- **影響**: Ledgeaの11カ国対応に直結する問題。後付けだと全コンポーネントに手戻りが発生する
- **対策**: トークン設計段階でi18n（国際化）を前提条件に含める
- **ルール化**: design-system.md にASEAN多言語フォーマット定義を追加する

## 学習3: 汎用ルールとプロダクト固有ルールの分離
- **状況**: ui-design-patterns.md が「汎用モバイルアプリのベストプラクティス集」になっていた
- **影響**: Ledgea固有のUXパターン（多通貨表示、国別減価償却ルール切り替え等）が未定義
- **対策**: 汎用パターン（ui-design-patterns.md）とプロダクト固有パターン（例: ledgea-ui-patterns.md）の分離を検討する

## 学習4: brand-guardianのボトルネック化リスク
- **状況**: 3エージェント（ui-designer, motion-designer, whimsy-injector）の完了条件がbrand-guardianのレビュー承認に依存
- **影響**: brand-guardianに作業が集中し、パイプラインが詰まるリスク
- **対策**: SLA（48時間以内に返答なき場合は承認とみなす）を設定した
