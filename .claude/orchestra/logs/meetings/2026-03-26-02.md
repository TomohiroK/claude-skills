# CxOボードミーティング議事録

**日時**: 2026-03-26
**アジェンダ**: 歯科衛生士お友達紹介キャンペーンLP（one-dental-referral-lp）の制作振り返り

**記録**: secretary

## 各CxOの主要意見

### CMO（マーケティング）
- CTA設計は4段構え（ヘッダー/ヒーロー/中間/最終ステップ式）で良好
- 最終CTAの会員確認→分岐UIは秀逸。会員・非会員両方を取りこぼさない
- 「祈るだけ」のユーモア表現は共感を生むが、CTA内では行動促進が弱まるリスク
- **競合比較セクションが未搭載**。調査済みの差別化ポイント（登録のみで成立）を活かし切れていない
- 社会的証明（参加者の声、実績数値）が不足
- A/Bテスト設計・計測タグが未準備

### CLO（法務）
- 景表法対応は概ね良好。「役務の対価」構成が利用規約・LP注意事項に反映済み
- **[重大] 利用規約の期間（4/1〜5/31）がLP本体（5/1〜6/30）と不整合** → **即座に修正済み**
- **[重大] 法人名の不整合**: LP「株式会社VISTA」vs 利用規約「VISTA株式会社」→ 正式名称を依頼者に確認必要
- 勤続お祝い金の主語（歯科医院が支給）は適切に処理されている
- 先着1,000名の判定基準・終了可能性・対象外の可能性は全て明示済み
- meta descriptionの「即成立」は審査プロセスがある以上不正確 → 修正推奨
- 職安法リスクの外部弁護士確認が未完了

### CTO（技術）
- HTML構造・セマンティクス・SEO対応は高水準
- **[最重要] Tailwind CDNは開発専用。本番前にビルドCSSに置き換え必須**
- `</br>` の誤記（2箇所）を修正すべき
- FAQの`max-height: 300px`は動的計算に変更推奨
- JS内のTailwindクラス名ハードコードは保守性低下の原因
- Font Awesomeフル読み込み（~70KB）→ サブセット化推奨
- 一部装飾要素にpointer-events-none漏れあり

### CVO（ビジュアル）
- **品質ゲート不合格**: ヒーロー実画像が未実装、ライフスタイルバナーが画像なし
- ヘッダーのグラデーション色がヒーローと微妙にずれている → 統一すべき
- ゴールドCTA + text-gray-900のコントラストは合格
- ステップUIのインタラクションデザインは優秀
- OGP画像が未設定 → SNSシェア時にビジュアルなし
- 未圧縮画像（hero-original.png 8.5MB, image1.png 8.1MB）がデプロイ対象に残存

### COO（執行）
- 手戻りの主因: 要件確認の粒度不足（用語3回反復、ヘッダー後付け調整）
- CMOレビューが実装後だった → ワイヤーフレーム段階で実施すべき
- campaign-lp.mdルール（CLO/CVO早期参加）の遵守が不十分
- ボタン改行問題: MEMORYにルール化済みなのに再発 → `whitespace-nowrap`をデフォルト適用する構造的対策を提案

## 学習事項

1. **要件確認の粒度**: ユーザー向けラベル・用語は実装前に合意する。仮定での実装は手戻りコストが質問コストの数倍
2. **CxOレビューのタイミング**: 実装後レビューでは手戻りが大きい。ワイヤーフレーム段階でCMO/CLO/CVOの並列レビューを実施する
3. **3点セットの同時管理**: LP本体・ended.html・利用規約のデータ（期間、名称、条件）は同一ソースから生成し、不整合を構造的に防止する
4. **ボタン改行の構造的防止**: `whitespace-nowrap`をデフォルト適用。ルールの存在と実行のギャップを構造で埋める
5. **CDN版Tailwindは開発限定**: 本番デプロイ前のビルドパイプライン工程をチェックリストに含める
6. **金銭キャンペーンのフィッシング対策**: 注意喚起をLP本体と終了ページの両方に配置する（終了後こそ狙われる）

## 総合評価

- **構造・導線・法務対応**: 高水準。campaign-lp.mdルールが機能している
- **プロセス**: 要件確認不足による手戻りが最大の課題。ルール化で次回改善見込み
- **ビジュアル**: CVO品質ゲート不合格（実画像未実装）。公開前に必ず対応
- **技術**: Tailwind CDNの本番利用が最大リスク。ビルドパイプライン整備が必須
