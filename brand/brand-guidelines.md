### 1.5 ブランドパーソナリティ

Ledgea のブランドパーソナリティは、以下の 5 つのキーワードで定義される。コンテンツ・デザイン・コミュニケーションの判断軸として使用すること。

#### Precise（精緻）
数値・制度・データに対して妥協しない正確さ。小数点、税率、減価償却の計算式——細部への誠実さがユーザーの信頼を生む。曖昧な表現や「おおよそ」の言い回しはブランドの誠実さを損なう。

#### Fluid（流動的）
ASEAN 各国の制度変更、為替変動、組織の成長に柔軟に追随する適応性。硬直した既存システムへのアンチテーゼとして、変化を前提に設計されている姿勢を示す。

#### Rooted（根ざした）
アジアの商習慣・文化・法制度への深い理解。表面的なグローバル対応ではなく、各国の「現場」を知っていることへの誇り。現地の声から生まれたプロダクトであることを体現する。

#### Lucid（明快）
複雑な制度を、使う人にとって分かりやすく伝える翻訳者としての役割。専門用語の羅列でなく、意味が直感的に伝わるインターフェースとコミュニケーション。

#### Enduring（永続的）
一時的なトレンドに流されない堅牢さと長期的信頼。データ保全、コンプライアンス対応、監査への強さを示す。「使い捨て」でなく「資産」としてのソフトウェアという哲学。

### 1.6 ブランドボイス＆トーン

#### ボイス（変わらない声の質）

Ledgea は常に以下の声の質を保つ。

- **誠実で直接的** — 過剰な修飾語を使わず、事実と根拠で話す
- **知識豊富だが押しつけがましくない** — 専門性を見せびらかさず、ユーザーの判断を助ける情報を提供する
- **温かみがあるが軽薄でない** — 人間的な親しみやすさと、業務ツールとしての品位を両立する
- **謙虚だが確信を持っている** — 不確かなことは明言し、確かなことははっきり伝える

#### トーン（文脈によって変わる声の色）

| 文脈 | トーン | 具体例 |
|------|--------|--------|
| マーケティング・LP | 明快・自信 | "Your fixed assets. Every country. One system." |
| オンボーディング | 親切・ガイド的 | "まず、日本法人の会計年度を設定しましょう。" |
| エラーメッセージ | 冷静・解決志向 | "取得原価の入力形式を確認してください（例: 1,500,000）。" |
| リリースノート・変更通知 | 正確・簡潔 | "タイ歳入法 Section 65 改正に対応（2025 年 10 月施行）。" |
| サポートコミュニケーション | 共感的・実務的 | "ご不便をおかけし申し訳ございません。担当者が確認いたします。" |

## 2. カラーシステム

### 2.1 プライマリカラー

#### Teal — Ledgea Core

```
HEX:  #0D9488
RGB:  13, 148, 136
HSL:  177°, 84%, 32%
CMYK: 91, 0, 8, 42
```

ブランドの中心色。ロゴ、主要 CTA、重要 UI コンポーネントに使用する。Teal は「精緻さ（Precise）」と「流動性（Fluid）」を視覚的に体現する色として選定されている。

### 2.2 セカンダリカラー

#### Sky Blue — Interactive / Accent

```
HEX:  #0EA5E9
RGB:  14, 165, 233
HSL:  199°, 89%, 48%
CMYK: 94, 29, 0, 9
```

リンク、インタラクティブ要素、ハイライトに使用する。Teal との組み合わせで使用し、単独での大面積使用は避ける。

### 2.6 アクセシビリティ要件

- すべてのテキストと背景の組み合わせは WCAG 2.1 AA 基準（コントラスト比 4.5:1 以上）を満たすこと
- Teal (#0D9488) on White: コントラスト比 **4.54:1** — AA 合格
- Teal (#0D9488) on `neutral-50`: コントラスト比 **4.41:1** — AA 合格（Large Text のみ）
  - **注意:** `neutral-50` 背景上の小サイズ本文テキストに Teal を使用する場合は `neutral-700` (#374151) を使用すること
- Sky Blue (#0EA5E9) on White: コントラスト比 **2.96:1** — **AA 不合格** — 装飾目的のみに限定し、テキストカラーとして単独使用禁止

### 2.7 禁止カラー使用

- このガイドラインに定義されていないカラーコードを新規採用することは禁止
- 追加カラーが必要な場合はブランドガーディアンへ申請し、ガイドライン改訂プロセスを経ること
- 文化的配慮から特定文脈での制限があるカラーについては [第 5 章](#5-文化的配慮ガイドラインasean特有) を参照

## 3. タイポグラフィ

### 3.1 フォントファミリー

| 役割 | フォント | 理由 |
|------|---------|------|
| ロゴ専用 | Plus Jakarta Sans | 幾何学的でありながら温かみのある造形。アジア市場での視認性と現代性を両立 |
| 見出し | Space Grotesk | 技術的精緻さと個性を持つ。数字の視認性が高くフィンテック文脈に最適 |
| 本文 | Inter | 高い可読性と多言語サポート。長文・UI テキスト・小サイズでの安定した表示 |

### 3.4 日本語フォント対応

本文・UI テキストが日本語を含む場合、以下のフォントスタックを使用する。

```css
/* 日本語本文 */
font-family: 'Inter', 'Noto Sans JP', 'Hiragino Sans', 'Yu Gothic UI',
             'Meiryo UI', sans-serif;

/* 日本語見出し */
font-family: 'Space Grotesk', 'Noto Sans JP', 'Hiragino Sans', sans-serif;
```

- Noto Sans JP をフォールバックとして採用する（Google Fonts で提供）
- 日本語テキストの Line-height は英数字より 0.1〜0.2 大きく設定する（最低 1.8 を推奨）
- 日本語 Caption（12px）はアンチエイリアス問題のため 13px 以上を推奨

### 3.5 タイポグラフィ禁止事項

- Plus Jakarta Sans はロゴ以外の用途に使用しない
- 見出しに Inter を使用しない（Space Grotesk を使用すること）
- ウェイト 300 Light を 14px 未満で使用しない
- 文字間隔（letter-spacing）を -0.05em 未満に設定しない
- 行間（line-height）を 1.0 未満に設定しない

## 4. ロゴ使用ルール

### 4.1 ロゴバリエーション

| バリエーション | 用途 | 背景 |
|--------------|------|------|
| フルカラー（Teal on White） | 標準使用。デジタル・印刷全般 | 白・ライトニュートラル |
| フルカラー（White on Teal） | ダーク背景・Teal 背景上 | ダーク・Teal |
| モノクローム（Black） | 白黒印刷・FAX・刻印 | 白・ライトグレー |
| モノクローム（White） | 暗色背景上での白黒印刷 | ダーク背景 |

### 4.2 最小サイズ

| 媒体 | 最小幅 |
|------|--------|
| デジタル（画面表示） | 120px |
| 印刷 | 25mm |
| ファビコン / アプリアイコン | 16px（シンボルマークのみ使用） |

最小サイズを下回る場合、ロゴワードマークの使用は禁止。シンボルマーク（Ledgea のマークのみ）または代替テキスト表記を使用すること。

### 4.3 クリアスペース

ロゴの周囲には、ロゴの高さ（H）の 0.5 倍以上のクリアスペースを確保すること。

```
      ← 0.5H →
  ┌───────────────┐
  │               │ ↑ 0.5H
  │  [L] Ledgea   │
  │               │ ↓ 0.5H
  └───────────────┘
      ← 0.5H →
```

クリアスペース内にテキスト、図形、写真、他のロゴを配置しない。

### 4.4 禁止事項（NG 例）

以下の使用はいかなる場合も禁止する。

| # | 禁止内容 |
|---|---------|
| 1 | ロゴの縦横比を変更する（縦長・横長に変形） |
| 2 | ロゴに影（ドロップシャドウ）、グロー、グラデーションを適用する |
| 3 | ガイドライン外のカラーでロゴを表示する |
| 4 | ロゴの一部を切り取る、マスクをかける |
| 5 | ロゴを回転・傾斜させる |
| 6 | ロゴ文字を別フォントで再現する |
| 7 | ロゴにアウトラインのみの表示（塗りなし）を使用する |
| 8 | ロゴ上に他のテキストや要素を重ねる |
| 9 | 視認性の低い背景（複雑なパターン、低コントラスト写真）上に配置する |
| 10 | 競合製品のロゴと並列・比較レイアウトに使用する |

### 4.5 ロゴ使用承認フロー

サードパーティがロゴを使用する場合（パートナー資料・メディア掲載等）は、使用前に必ずブランドガーディアンへ申請し書面による承認を得ること。未承認使用は法的対応の対象となる。

## 5. 文化的配慮ガイドライン（ASEAN特有）

> **重要:** このセクションに記載された事項は単なる推奨ではなく、法的リスク・宗教的配慮・商習慣に基づく必須要件を含む。法的リスクが疑われる表現は、ブランドガーディアンおよび法務部門へ即時エスカレーションすること。

### 5.1 全地域共通

#### 数字「4」の扱い

「4」は中国語・日本語・韓国語圏で「死」と同音異義のため、縁起が悪いとされる。

- プロダクト機能の命名（例: Plan 4、Version 4.0 の強調）は可能な限り回避または代替表現を検討する
- 価格プランは「1, 2, 3, 5」のステップを優先し、「4」を主要プランの番号として使用しない
- ただし、GAAP 上の数値・会計数値・法令引用における「4」の使用は通常どおり行う

#### 赤色の扱い

- 中国・ベトナム・タイなどでは赤は「幸運・吉祥」を意味する。エラー表示の赤をデザインの主役にしないよう注意する
- エラーや警告に赤を使う場合は、アイコンや文言で補足し、赤のみで意味を伝えない

### 5.2 日本

#### 敬語・丁寧表現

- BtoB SaaS の文脈では「です・ます体」を基本とし、マーケティングと UI 双方で統一すること
- 「様」「御社」「貴社」の使い分けを誤らない（書面では「貴社」、会話では「御社」）
- UI 内での命令形（「入力せよ」「クリックしろ」）は絶対に使用しない

#### 色の文化的意味（日本）

- 白は「清潔・誠実」を象徴し、ビジネス文書に適切
- 黒は「信頼・高級感」に結びつく。過度な使用は「喪」を連想させる文脈を避ける
- 金色は「高品質」を意味するが、過剰使用は派手な印象を与えるため Ledgea では使用しない

### 5.3 タイ

#### 仏教・王室への配慮（Critical — 法的リスク）

- **仏像・仏陀の画像を商業目的に使用することは厳禁。** タイ法律（刑法・宗教法）違反となる可能性があり、ブランドへの致命的ダメージをもたらす
- 王室（国王・王族）の画像・名前の商業利用は不敬罪（Lèse-majesté, 刑法第 112 条）に抵触する可能性がある。使用前に法務部門の確認を必須とする
- 頭を触る行為を示す画像（インフォグラフィック・イラスト等）は避ける（頭は神聖とされる）
- 足を指し示す画像・足で物を指す表現は避ける（足は不浄とされる）

#### 色の文化的意味（タイ）

- 黄色は王室の色。単独での大面積使用は王室・仏教への言及と受け取られる可能性がある。マーケティング用途での黄色の主役使用は避ける
- 白は喪に服す色と認識されることがある文脈に注意する（葬儀関連のコミュニケーションと混同しない）

### 5.4 マレーシア

#### 宗教的配慮（イスラム教）

- 豚・犬のイラストや写真は避ける（イスラム教では不浄とされる）
- アルコールをポジティブに描写するビジュアルは避ける
- ラマダン期間中のコミュニケーションは断食・礼拝への配慮を示す文言を検討する

#### 色の文化的意味（マレーシア）

- **黄色の単独大面積使用は回避する。** マレーシアでは黄色は王室の色であり、政治的・宗教的文脈と混同される可能性がある
- 緑はイスラム教の神聖な色として認識される。宗教的文脈での誤用に注意する

#### 多民族・多言語配慮

- マレーシアはマレー系・中華系・インド系の多民族国家。特定民族のみを前面に出すビジュアルは避け、多様性を反映する
- 公式コミュニケーションはマレー語（Bahasa Malaysia）を基本とし、必要に応じて英語・中国語（繁体字）を併記する

### 5.5 インドネシア

#### 宗教的配慮（イスラム教 — 世界最大のイスラム教徒人口を擁する国）

- マレーシアと同様、豚・犬の描写、アルコールのポジティブ描写を避ける
- ラマダン（断食月）期間中は特に配慮したコミュニケーションを実施する
- 「ハラール（Halal）」認証に言及する際は正確な根拠を示すこと。根拠のないハラール表示は景品表示法等に抵触する可能性がある（**法務エスカレーション対象**）

#### パンチャシラ（建国五原則）への配慮

- 特定宗教を優遇・排除するような表現は、インドネシアの国是であるパンチャシラ（宗教的多元主義等）に反するとみなされる可能性がある

#### 色の文化的意味（インドネシア）

- 赤と白はインドネシア国旗の色。この組み合わせを国旗と誤認させるような使用は避ける
- 緑はイスラム教と結びつく聖なる色。不用意な使用は宗教的含意を持つ可能性がある

### 5.6 ベトナム

#### 色の文化的意味（ベトナム）

- 赤と黄色はベトナム国旗の色。この組み合わせを国旗と誤認させるような使用は避ける
- 黒は死・喪を連想させる。ビジネスコミュニケーションでの大面積使用は避ける
- 緑と白の組み合わせは一般的に清潔・爽やかを意味し、ポジティブに受け取られる

#### 歴史的・政治的配慮

- 南北ベトナム分断に関連する表現・地図表記は慎重に扱う
- 地図を使用する場合、ベトナム社会主義共和国の公式国境線を使用すること（**法的リスク対象**）

### 5.7 シンガポール / その他 ASEAN

- シンガポールは多民族・多宗教国家（中華系・マレー系・インド系）であり、マレーシア同様の多様性配慮が必要
- フィリピンはカトリックが主流。クリスマス等の宗教行事は重要なコミュニケーション機会だが、宗教的シンボルの商業利用は慎重に扱う
- ミャンマー・カンボジアは仏教が国民文化の根幹。タイと同様の仏教関連配慮を適用する

## 6. ライティングガイドライン

### 6.1 日本語トーン

#### 基本方針

- **文体:** 敬体（です・ます体）を基本とする。常体（だ・である体）はブログ記事等で採用可能だが、製品 UI・公式コミュニケーションでは使用しない
- **語調:** 丁寧・明確・簡潔。過剰な謙遜表現（「つたない…」「粗品ですが…」）はビジネス SaaS には不適切
- **文章の長さ:** 1 文を 40〜60 文字以内に収めることを目標とする。長い文はピリオドで分割する

#### OK 表現 / NG 表現 対比

| # | NG 表現 | OK 表現 | 理由 |
|---|---------|---------|------|
| 1 | 「弊社のソリューションは貴社の課題をソルブします」 | 「固定資産管理の課題を解決します」 | カタカナ語の不必要な使用を避ける |
| 2 | 「簡単・カンタン・かんたんに使えます！」 | 「直感的に操作できます」 | 過剰な強調・表記揺れを避ける |
| 3 | 「登録してください。入力してください。押してください。」 | 「会計年度を登録して、次へ進みましょう。」 | 命令の連続をガイダンスに転換する |
| 4 | 「エラーが発生しました」 | 「日付の形式を確認してください（例: 2025/04/01）。」 | エラーは原因と解決策をセットで伝える |
| 5 | 「最強の固定資産管理ツール」 | 「ASEAN 11カ国に対応した固定資産管理 SaaS」 | 根拠のない最上級表現は景表法リスクあり（法務確認必須） |
| 6 | 「〇〇社様との比較で2倍速い！」 | 「処理速度の詳細はお問い合わせください。」 | 比較広告は根拠の法的確認なしに使用禁止 |
| 7 | 「ご入力頂けますでしょうか？」 | 「取得原価をご入力ください。」 | 過剰な敬語は読みにくく、UI の明快さを損なう |
| 8 | 「タイ・インドネシア・ベトナム・フィリピン・マレーシア・シンガポール・ミャンマー・カンボジア・ラオス・ブルネイ・東ティモール対応！」 | 「ASEAN 11カ国 + 日本対応」 | 列挙よりも統括表現を優先する |
| 9 | 「データが消えることはありません！」 | 「データは AES-256 暗号化で保護されます。」 | 根拠のない断言は禁止。技術的根拠を示す |
| 10 | 「いつでもどこでも使えます」 | 「ブラウザがあればオフライン環境でも動作します（WebAssembly 採用）。」 | 曖昧な便益を技術的事実で裏付ける |

### 6.2 英語トーン

#### 基本方針

- **文体:** Active voice を基本とする。Passive voice は技術文書・法的文書のみ許容
- **語調:** Professional but approachable. Avoid jargon unless the audience is confirmed to be technical
- **文章の長さ:** Aim for sentences under 20 words. Use bullet points for instructions with 3+ steps

#### OK 表現 / NG 表現 対比

| # | NG 表現 | OK 表現 | 理由 |
|---|---------|---------|------|
| 1 | "Utilizing our innovative, cutting-edge solution…" | "Ledgea handles fixed asset compliance across ASEAN." | Eliminate filler superlatives |
| 2 | "The system will be leveraged to…" | "Use Ledgea to…" | Prefer active, direct construction |
| 3 | "Our world-class, best-in-class platform…" | "Built for ASEAN's regulatory complexity." | Unverified superlatives require legal review |
| 4 | "Easy! Simple! Intuitive!" | "Set up your first asset register in under 10 minutes." | Replace adjectives with concrete claims |
| 5 | "Please kindly input the data." | "Enter your acquisition cost." | Remove redundant politeness markers |
| 6 | "We support Thailand, Indonesia, Vietnam, Philippines, Malaysia, Singapore, Myanmar, Cambodia, Laos, Brunei, Timor-Leste and Japan." | "Supporting ASEAN 11 countries and Japan." | Prioritize concise coverage statements |
| 7 | "100% uptime guaranteed!" | "99.9% uptime SLA. See terms for details." | Absolute guarantees require legal sign-off |
| 8 | "Compared to [competitor], 2x faster." | "Performance benchmarks available on request." | Comparative advertising requires verified data |
| 9 | "Your data is totally safe with us!" | "Data encrypted with AES-256 at rest and in transit." | Vague assurances undermine trust; use specifics |
| 10 | "Asia-friendly accounting." | "Compliant with tax regulations across ASEAN 11 + Japan." | "Asia-friendly" is vague; state what compliance means |

### 6.3 専門用語の扱い

#### 会計・税務用語

| 日本語用語 | 英語用語 | 使用方針 |
|-----------|---------|---------|
| 固定資産 | Fixed Asset | UI・文書で統一使用 |
| 減価償却 | Depreciation | 用語統一。「償却」単独使用は文脈依存のため要注意 |
| 取得原価 | Acquisition Cost | UI ラベルに使用 |
| 帳簿価額 | Book Value | 「残存価値」との混同に注意（Residual Value は別概念） |
| 耐用年数 | Useful Life | 法定耐用年数（Statutory Useful Life）と区別して使用 |
| 償却方法 | Depreciation Method | 定額法（Straight-line）、定率法（Declining Balance）を明記 |
| 台帳 | Ledger / Register | コンテキストにより使い分け。製品名 Ledgea との混同に注意 |

#### 地域別法令用語

- 各国の法令用語は現地語または広く認知された英語表記を使用する
- 日本: 法人税法、中小企業特例、グループ通算制度
- タイ: Revenue Code Section 65, BOI Privileges
- インドネシア: PPh 11/2021, Peraturan Pemerintah
- ベトナム: Circular 45/2013/TT-BTC, Circular 147/2016/TT-BTC

### 6.4 NGワード・使用禁止表現

以下の表現は、ブランド観点・法的リスク・文化的配慮から使用を禁止する。

#### 法的リスク（要法務確認）

| NGワード / 表現 | 理由 |
|--------------|------|
| 「完全無料」「永久無料」 | 景表法：条件の明記なしに使用禁止 |
| 「業界最速」「日本一」「No.1」 | 景表法：客観的根拠のない最上級表現禁止 |
| 「〇〇社との比較で…」 | 不正競争防止法：比較広告は根拠と法務確認が必要 |
| 「ハラール対応」 | 根拠のない宗教認証表示は詐称広告リスク |
| 「医療・薬事関連の効能表記」 | 薬機法（本プロダクトに直接関係しないが一般原則として記載） |
| 「保証します」「絶対に」 | 根拠のない断定的表示は景表法リスク |

#### ブランドボイス逸脱

| NGワード / 表現 | 理由 |
|--------------|------|
| 「革命的」「ゲームチェンジャー」 | 過剰表現。Precise・Lucid ブランド価値と不整合 |
| 「簡単すぎる」「誰でも使える」 | 曖昧な便益。具体的な根拠に置き換える |
| 「最先端」「次世代」 | 根拠不明の形容詞。技術的事実で置き換える |
| 「アジアフレンドリー」 | 曖昧。具体的な対応国・法令を明示する |
| 「グローバルスタンダード」 | 実態と乖離する可能性。使用する場合は根拠を明記 |

#### 文化的配慮

| NGワード / 表現 | 理由 |
|--------------|------|
| 「東南アジア」をひとまとめにした表現 | 各国の制度・文化の独自性を軽視する |
| 「発展途上国」 | 差別的。「新興市場」「成長市場」を使用する |
| 「アジア人は…」等の集団的表現 | 多様性を無視したステレオタイプ表現 |
| 宗教・民族を否定的文脈で使用 | ブランドとして絶対に許容しない |

## 付記: 今回のレビューで発見された新規ルール候補

以下は今回のガイドライン策定プロセスで新たに言語化された事項。正式なガイドライン条項として追加するには、プロジェクトオーナーの承認を要する。

1. **Sky Blue (#0EA5E9) のテキスト使用禁止:** アクセシビリティ検証の結果、White 背景での Sky Blue テキストはコントラスト比 2.96:1 と AA 基準未達。装飾目的のみに限定するルールを明文化する
2. **数字「4」回避オプション:** 価格プランの番号付けにおいて「4」を回避するオプションを価格・プロダクトチームと共有し、採用可否を決定する
3. **タイ王室・仏教関連コンテンツの事前法務確認ルール:** 該当コンテンツの制作前に法務部門の承認を必須とするフローを正式化する
4. **比較広告の全件法務確認ルール:** 競合他社や自社旧バージョンとの比較表現は、全件マーケティング責任者 + 法務部門の事前承認を必須とする
5. **地図使用時の国境線確認ルール:** ベトナム・中国・南シナ海等の領土紛争地域を含む地図を使用する際は、公式政府機関の地図データを参照し法務確認を行うルールを追加する
